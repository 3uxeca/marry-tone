무료 오픈소스만 사용해서 얼굴 이미지 1장을 입력받으면 피부색과 눈동자색을 분석하고, 그 결과를 기반으로 PCCS 색채계 또는 KS 색채계에 매핑된 결과를 반환하는 MVP용 분석 시스템의 기술 블루프린트를 설계해줘.

내가 원하는 범위는 “퍼스널컬러 4계절 진단” 전체가 아니라, 아래까지만 우선 구현 가능한 구조다.

[목표]
- 업로드된 얼굴 이미지에서
  1) 얼굴 검출
  2) 피부 영역 추출
  3) 눈동자/홍채 영역 추출
  4) 대표 색상 추출
  5) 추출 색상을 PCCS 또는 KS 색채계 기준값으로 매핑
  6) JSON 결과 반환
- 결과는 진단명(봄웜/여쿨 같은 시즌명) 없이도 됨
- 핵심은 “피부/눈동자 대표색 → 표준 색체계 결과” 도출이다

[중요 제약]
- 반드시 무료 오픈소스만 사용
- 상용 API, 유료 SaaS, 클라우드 비전 API 사용 금지
- 구현 난이도와 현실성을 우선
- 학술적으로 완벽한 정확도보다 MVP 수준의 재현 가능한 파이프라인이 중요
- API 형태로 붙일 수 있어야 함
- 입력은 정면에 가까운 단일 인물 얼굴 사진 1장 기준
- 출력은 JSON
- Python 기반 백엔드 API(FastAPI 우선) 기준으로 설계

[원하는 산출물]
1. 사용할 수 있는 무료 오픈소스 후보 정리
   - 얼굴 검출
   - 얼굴 랜드마크
   - 피부 영역 분리
   - 홍채/눈 영역 추출
   - 대표 색 추출
   - 색공간 변환(RGB/Lab/HSV)
2. 각 후보의 장단점 비교
3. 가장 현실적인 조합 1개 추천
4. 전체 처리 파이프라인 설계
5. API 엔드포인트 설계
6. JSON 응답 스키마 예시
7. 정확도 이슈 / 조명 이슈 / 화장 / 렌즈 착용 등 실패 케이스 정리
8. 향후 시즌 진단(퍼스널컬러)로 확장하는 방법
9. 반드시 실제 구현 가능한 폴더 구조까지 제안

[기술 선호]
- Python
- FastAPI
- OpenCV
- MediaPipe / dlib / face-alignment 류 허용
- segmentation이나 landmark용 공개 모델 허용
- 색상 추출은 K-means, median, dominant color 등 단순하고 재현 가능한 방식 선호

[색채계 요구사항]
- PCCS 또는 KS 색채계로 최종 매핑하는 로직이 필요함
- 완벽한 공식 데이터가 없으면, 우선 내부 기준 테이블을 설계해서 nearest color 매핑 방식으로 구현 가능하게 해줘
- 즉 “대표색 추출 → Lab 변환 → 기준 팔레트와 거리 계산 → 가장 가까운 PCCS/KS 코드 반환” 구조여도 괜찮다

[꼭 포함할 것]

- 왜 이 조합이 MVP에 적합한지
- 오픈소스 라이선스 확인 포인트
- 실제 구현 순서
- 1차 MVP / 2차 개선 단계 구분

- 불필요한 마케팅 설명
